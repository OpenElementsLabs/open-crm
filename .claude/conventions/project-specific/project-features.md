# Project Features

Open CRM is an open-source CRM system designed for startups and small teams to manage business relationships. It focuses on core CRM needs — companies, contacts, and communication history — without the complexity of enterprise solutions. The project is in early development.

## Core Features

- **Company Management** — Create, view, edit, and soft-delete companies with name, email, website, phone number, and address details (street, house number, zip code, city, country). Supports filtering by name and Brevo origin, pagination, and restoring archived companies. The detail view shows the address as a formatted multi-line block. The list shows contact count columns with edit and comment action buttons per row.
- **Contact Management** — Create, view, edit, and hard-delete contacts with name, email, phone, position, gender, language, LinkedIn URL, and optional birthday. Contacts can be linked to a company via dropdown. The language field is optional and displayed as a translated label (e.g., "Deutsch"/"Englisch" in DE UI). Supports unified search across name, email, and company name, plus filters for company (including "No company" for unassigned contacts), language, and Brevo origin. Birthday is displayed in locale-specific format (DE: DD.MM.YYYY, EN: MM/DD/YYYY). Bidirectional navigation links between contact and company detail views.
- **Detail Field Actions** — Detail view fields have interactive action icons (always visible in light color, darkening on hover, full color on touch devices): copy-to-clipboard for email/website/phone/LinkedIn/address, open-in-new-tab for URLs (website, LinkedIn with automatic https:// normalization), mailto: for email fields, and tel: for phone fields. Copy shows a brief checkmark feedback.
- **Comments** — Add comments to companies and contacts via a modal dialog. Author is set to the authenticated user's name from the OIDC token. Comments are displayed with pagination ("Load more"), sorted by creation date descending. The comment count in the heading updates live after adding a comment.
- **Image Upload** — Upload, display, and remove company logos (SVG, PNG, JPEG) and contact photos (JPEG only). Thumbnails appear in list tables (32x32); larger images display in detail views. Images are stored as `bytea` in PostgreSQL with a 2 MB size limit. DTOs expose `hasLogo`/`hasPhoto` boolean flags.
- **CSV Export** — Export filtered company or contact data as CSV files. A modal dialog with checkboxes allows selecting which columns to include (all available by default). CSV is generated on the backend using Apache Commons CSV with UTF-8 BOM encoding, comma separator, and technical field names as headers. Button is disabled when no results match the current filter.
- **Print Views** — Print button opens a new tab with all filtered records in a print-optimized table. Optimized for DIN A4 portrait with text wrapping, repeating table headers on page breaks, and Safari compatibility. Comment count column is excluded from print.
- **Brevo Import** — One-directional import from Brevo to CRM, triggered manually from a dedicated "Brevo Import" page. Imports companies (name, domain) and contacts (name, email, phone, position, LinkedIn, language) with field mapping. Matches by Brevo ID on subsequent imports, or by name/email on first import. Brevo-managed fields on contacts (firstName, lastName, email, language) are read-only. Re-import only overwrites Brevo-managed fields, preserving user-editable fields. A "Brevo" badge appears on imported records in detail views. Filtering by Brevo origin (All/From Brevo/Not from Brevo) is available on both lists.
- **Health Monitoring** — A health endpoint to verify backend availability, displayed in the frontend.
- **Internationalization (i18n)** — Frontend supports German and English with automatic language detection and manual switching via sidebar toggle.
- **API Documentation** — Swagger UI and OpenAPI spec auto-generated from backend annotations with @Parameter descriptions on all controller parameters.
- **OE Branding** — Sidebar header shows the app identity (placeholder logo + "Open CRM") and developer branding ("Developed by" + Open Elements logo linking to open-elements.com).
- **User Section** — Sidebar footer shows the authenticated user's name and profile picture (from OIDC token) with a working logout button that terminates both the frontend session and the OIDC provider session.
- **OIDC Authentication** — Full-stack OIDC authentication. Frontend: All pages require authentication via Auth.js v5 with automatic token refresh. API calls include JWT Bearer tokens via a server-side proxy route. Backend: Spring Security OAuth2 Resource Server validates JWT tokens on all API endpoints (except health and Swagger UI). User identity is extracted from JWT claims for comment authoring. Swagger UI includes an Authorize button for testing protected endpoints.
