# Project Features

Open CRM is an open-source CRM system designed for startups and small teams to manage business relationships. It focuses on core CRM needs — companies, contacts, tasks, and communication history — without the complexity of enterprise solutions. The project is in early development.

## Core Features

- **Company Management** — Create, view, edit, and delete companies with name, email, website, phone number, description, and address details (street, house number, zip code, city, country). Deletion is a hard delete with a confirmation dialog; contacts linked to a company block deletion (409 Conflict). Supports filtering by name, tags, and Brevo origin, pagination, and CSV export. The detail view shows the address as a formatted multi-line block. The list shows contact count columns with edit and comment action buttons per row.
- **Contact Management** — Create, view, edit, and hard-delete contacts with title, name, email, phone, position, gender, language, LinkedIn URL, description, and optional birthday. Contacts can be linked to a company via dropdown. The language field is optional and displayed as a translated label (e.g., "Deutsch"/"Englisch" in DE UI). Supports unified search across name, email, and company name, plus filters for company (including "No company" for unassigned contacts), language, tags, and Brevo origin. Birthday is displayed in locale-specific format (DE: DD.MM.YYYY, EN: MM/DD/YYYY). Bidirectional navigation links between contact and company detail views. Newsletter subscription status is tracked and synced from Brevo.
- **Task Management** — Create, view, edit, and delete tasks with an action description, due date, and status (Open, In Progress, Done). Tasks can be linked to a company and/or contact. Tags can be assigned to tasks via multi-select. Tasks support comments. The list view supports pagination and filtering. Detail view shows linked entities with navigation.
- **Detail Field Actions** — Detail view fields have interactive action icons (always visible in light color, darkening on hover, full color on touch devices): copy-to-clipboard for email/website/phone/LinkedIn/address, open-in-new-tab for URLs (website, LinkedIn with automatic https:// normalization), mailto: for email fields, and tel: for phone fields. Copy shows a brief checkmark feedback.
- **Comments** — Add, edit, and delete comments on companies, contacts, and tasks via a modal dialog. Author is set to the authenticated user's name from the OIDC token. Comments are displayed with pagination ("Load more"), sorted by creation date descending. The comment count in the heading updates live after adding a comment.
- **Image Upload** — Upload, display, and remove company logos (SVG, PNG, JPEG) and contact photos (JPEG only). Thumbnails appear in list tables (32x32); larger images display in detail views. Images are stored as `bytea` in PostgreSQL with a 2 MB size limit. DTOs expose `hasLogo`/`hasPhoto` boolean flags.
- **Tags** — Full-stack tag management with CRUD operations (name, description, color). Tags are shared between companies, contacts, and tasks via many-to-many relationships. Frontend includes a dedicated tag list page with search/pagination and live company/contact usage counts per tag (with navigation arrows to filtered entity lists), create/edit pages with a color picker (12 predefined colors + custom hex), colored tag chips on detail views, and a multi-select dropdown in entity edit forms and list page filters. Tag filter on lists supports AND semantics with deep-linking via URL query parameters. Tag assignment uses null-safe semantics (null = preserve existing, empty = remove all). Cascade delete on tag/entity deletion. Name uniqueness enforced.
- **CSV Export** — Export filtered company or contact data as CSV files. A modal dialog with checkboxes allows selecting which columns to include (all available by default). CSV is generated on the backend using Apache Commons CSV with UTF-8 BOM encoding, comma separator, and technical field names as headers. Button is disabled when no results match the current filter.
- **Print Views** — Print button opens a new tab with all filtered records in a print-optimized table. Optimized for DIN A4 portrait with text wrapping, repeating table headers on page breaks, and Safari compatibility. Comment count column is excluded from print.
- **Brevo Import** — One-directional import from Brevo to CRM, triggered manually from a dedicated "Brevo Import" page. Imports companies (name, domain) and contacts (name, email, phone, position, LinkedIn, language) with field mapping. Matches by Brevo ID on subsequent imports, or by name/email on first import. Brevo-managed fields on contacts (firstName, lastName, email, language) are read-only. Re-import only overwrites Brevo-managed fields, preserving user-editable fields. A "Brevo" badge appears on imported records in detail views. Filtering by Brevo origin (All/From Brevo/Not from Brevo) is available on both lists. Newsletter subscription status is synced from Brevo during import.
- **User Management** — OIDC users are stored in the database with sub (unique OIDC subject), name, email, and optional avatar. User info is synced from the OIDC token on login.
- **Admin Page** — An admin management page in the frontend for administrative functions.
- **Health Monitoring** — A health endpoint to verify backend availability, displayed in the frontend.
- **Internationalization (i18n)** — Frontend supports German and English with automatic language detection and manual switching via sidebar toggle.
- **API Documentation** — Swagger UI and OpenAPI spec auto-generated from backend annotations with @Parameter descriptions on all controller parameters.
- **OE Branding** — Sidebar header shows the app identity (placeholder logo + "Open CRM") and developer branding ("Developed by" + Open Elements logo linking to open-elements.com).
- **User Section** — Sidebar footer shows the authenticated user's name and profile picture (from OIDC token) with a working logout button that terminates both the frontend session and the OIDC provider session.
- **Custom Login Page** — Branded sign-in page replacing the default Auth.js page. Desktop: centered dark box (`#020144`) on light background with OE logo, "Open CRM" heading, login button, and "Developed by Open Elements" credit. Mobile: full-screen dark background. Displays a generic error message on authentication failure; technical details are logged to the browser console. Supports DE/EN via the existing i18n system.
- **OIDC Authentication** — Full-stack OIDC authentication. Frontend: All pages require authentication via Auth.js v5 with automatic token refresh. Unauthenticated users are redirected to the custom `/login` page. API calls include JWT Bearer tokens via a server-side proxy route. Backend: Spring Security OAuth2 Resource Server validates JWT tokens on all API endpoints (except health and Swagger UI which are public). User identity is extracted from JWT claims for comment authoring. Swagger UI includes an Authorize button for testing protected endpoints.
