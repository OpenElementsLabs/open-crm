This is primarily a **structural release**: the reusable parts of the
frontend (authentication, admin pages, translation UI) have been moved into
shared Open Elements libraries so they can power other applications, and the
admin area has been consolidated under a single `/admin/` URL prefix.
There are no new end-user features in this release.

---

## Highlights for Admins and Users

### Admin pages moved under `/admin/`

All admin-only pages now share a common URL prefix:

- API keys: `/admin/api-keys` (was `/api-keys`)
- Webhooks: `/admin/webhooks` (was `/webhooks`)
- Existing admin pages (users, audit log, server status, translation
  configuration, bearer token) are reached from the same group

The sidebar navigation has been updated to point at the new paths. Access
control is unchanged â€“ these pages remain gated by the `IT-ADMIN` role.

**If admins have bookmarked the previous `/api-keys` or `/webhooks` URLs,
those bookmarks need to be updated.** The old paths now return 404. Since
these pages are not exposed externally, this is an internal-only change.

### Translation buttons are now standalone components

The small **Translate** button (visible next to certain text fields when
the translation feature is configured) and its dialog are now reusable
components from the shared `@open-elements/ui` library. Behaviour is
unchanged for end users; the button still only appears when the translation
API is configured on the backend.

## Other changes under the hood

These changes are not directly visible to end users but may be useful for
admins to know:

- **New shared library: `@open-elements/nextjs-app-layer` 0.2.0.** The
  authentication factory, OIDC session handling, request proxy,
  middleware, and the admin pages (users, audit log, API keys, webhooks,
  server status, bearer token) have been extracted from Open CRM into a
  reusable Next.js library so future Open Elements applications can share
  the same foundation.
- **Two-step migration to the published package.** The library was first
  built as an in-repo workspace package, then published to npm; Open CRM
  now consumes it from npm. The `frontend/` project is back to a
  single-package layout (no more pnpm workspace), which simplifies
  `pnpm install` for contributors.
- **`@open-elements/ui` upgraded to 0.7.0.** Adds the new reusable
  `TranslateDialog` and `TranslateButton` components.
- **Wrapper files removed.** Several local re-export files
  (`session-provider.tsx`, `forbidden-page.tsx`, `add-comment-dialog.tsx`,
  `roles.ts`, `forbidden-error.ts`) have been removed â€“ imports now reach
  directly into `@open-elements/nextjs-app-layer`. Behaviour is identical.
- **API client interface standardised.** Admin pages now talk to the
  backend through a small, injectable API-client interface provided by the
  shared library. This is preparation for future improvements (test
  doubles, alternative backends) and has no user-visible impact.
