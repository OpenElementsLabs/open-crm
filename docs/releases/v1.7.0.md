# Open CRM v1.7.0 – Release Notes

**Released:** 2026-06-19
**GitHub release:** [v1.7.0](https://github.com/OpenElementsLabs/open-crm/releases/tag/v1.7.0)
**Previous version:** [v1.6.0](https://github.com/OpenElementsLabs/open-crm/releases/tag/v1.6.0) (2026-05-21)

This is the **largest feature release so far**. It introduces full-text
**global search** (powered by a new Meilisearch service), **bulk contact
import from CSV**, an **admin backup management page**, a first
integration with **Claude via the Model Context Protocol (MCP)**, and
broader support for **HEIC, WebP, and PNG contact photos**. It also
includes major dependency upgrades (Node.js 24, `spring-services` 1.0.0)
and one database migration.

Operators should read the **Upgrade notes** section carefully – this
release adds a new required service (Meilisearch) and changes the Docker
base image.

---

## Highlights for Admins and Users

### Global search across the CRM

A new **Search** entry has been added to the sidebar (right below
Updates). It opens a dedicated search page that searches across companies,
contacts, tags, and comments in one go:

- **Typo-tolerant** – e.g. `hendirk` still finds `Hendrik`
- **Grouped results** by entity type (Companies, Contacts, Tags, Comments)
- **Comment hits link back** to the company or contact they belong to
- Hits use **highlighted** snippets to show where the match was found
- Minimum of two characters required to avoid runaway queries
- Up to 5 hits per group by default, expandable up to 20

Search is powered by a new **Meilisearch** service that runs alongside
the backend (see Upgrade notes). The CRM (re-)indexes all data on every
backend startup, and keeps the index up to date as users add, edit, or
delete entities.

### Contact photos: HEIC, WebP, and PNG accepted

Contact photo upload has been broadened beyond JPEG. The server now
accepts **JPEG, PNG, WebP, and HEIC** (iPhone default) and transcodes
non-JPEG formats to JPEG on ingest:

- **HEIC** files from iPhones can be uploaded directly – no need to
  convert them first
- **WebP** images saved from the browser are accepted
- **PNG** uploads with transparent backgrounds are flattened against
  white during conversion
- **EXIF orientation** is honoured, so photos taken on a phone end up
  rotated upright in the CRM
- **EXIF/GPS metadata is stripped** during transcoding (privacy
  improvement)
- The upload size limit has been **raised from 2 MB to 20 MB** to
  accommodate higher-resolution mobile photos

If the server is missing the HEIC native library, HEIC uploads fail
gracefully with a clear "415 Unsupported Media Type" error; JPEG, PNG,
and WebP uploads continue to work in that case.

### Audit log: entity names alongside IDs

Audit log entries now carry the **name of the affected entity** (company
name, contact name, …) in addition to its UUID. Historical rows continue
to display, but show `UNKNOWN` for the name field. The new field is
already exposed by the backend; the admin UI will be updated to show it
in a follow-up release.

### Branded login screen

The login page now shows the application name **"Open CRM"** instead
of a generic "Login" heading. Small change, but it makes the SSO landing
look consistent with the rest of the application.

## Other changes under the hood

These changes are not directly visible to end users but may be useful for
admins to know:

- **`spring-services` upgraded to 1.0.0** (from 0.15.0). The upgrade
  brings shared image-transcoding code (HEIC / WebP / PNG → JPEG),
  shared backup client (`DbBackupClient`), entity-name support for the
  audit log, and is the first stable 1.x of the library.
- **Node.js upgraded to 24** for the frontend build, with the package
  manager pinned to `pnpm@11.3.0` and `pnpm-workspace.yaml` re-added to
  the Docker image for install consistency.
- **`@open-elements/nextjs-app-layer` upgraded to 0.5.0** and
  **`@open-elements/ui` to 0.8.0**, bringing the admin route metadata
  and UI primitives in line with the new admin pages added in this
  release.
- **Backend test suite migrated from H2 to PostgreSQL via
  Testcontainers.** Tests now run against the same PostgreSQL 17 used in
  production, with Flyway migrations executed at startup and
  `ddl-auto: validate` enforcing that the entity model matches the
  migration files. The H2 dependency has been removed entirely.
  Developers need a running Docker daemon to execute `mvn test`.
- **Search code split into reusable lib and CRM glue.** The Meilisearch
  client and bootstrap infrastructure now live in a dedicated
  `com.openelements.crm.search.lib` sub-package in preparation for
  extraction into `spring-services`. A new `SearchIndexBootstrapStep`
  SPI allows per-entity bootstrap steps to fail in isolation. The
  Spring property prefix was renamed from
  `openelements.search.meilisearch.*` to `openelements.meilisearch.*`
  (env vars `MEILI_*` are unchanged).
- **Backend startup resilience.** The backend now retries the
  Meilisearch connection on startup instead of failing fast.
- **Architecture Decision Records.** A new ADR for the Meilisearch
  client integration was added under `docs/adr/`.
- **Search-related imports refactored** to use the
  `spring.base.services` package, in step with the `spring-services`
  1.0.0 upgrade.

## Upgrade notes

This release has the largest set of operational changes of any release
so far. Please review carefully before upgrading.

### New required service: Meilisearch

Global search depends on a **new Meilisearch v1.10 service** that must
be deployed alongside the backend. The provided `docker-compose.yml`
already includes a `meilisearch` service with a persistent volume
(`meili-data`). Operators running their own deployments must add this
service to their stack.

New environment variables:

- `MEILI_MASTER_KEY` – required; the master key for Meilisearch
- `MEILI_HOST` – default `http://meilisearch:7700`
- `MEILI_INDEX_PREFIX` – default `crm_`
- `MEILI_ENV` – default `production` (set to `development` in the
  override file)

In production, the Meilisearch port is **not** exposed outside the
Compose network. Indexes are stored in `/meili_data` inside the
Meilisearch container; please include the `meili-data` volume in
backup strategies if search index state is considered worth preserving.
(Data is also fully re-indexed from the database on every backend
startup, so loss of the Meilisearch volume is recoverable.)

### Docker base image change

To support HEIC contact photos, the backend Docker base image has
changed from **Alpine** to **Debian** (`eclipse-temurin:21-jre`):

- Image size grows from ~80 MB (Alpine) to ~250 MB (Debian)
- The image now includes `libheif1` and `libheif-plugin-libde265` for
  HEIC decoding
- If HEIC support is missing in a custom image, HEIC uploads return
  HTTP 415 and the other photo formats continue working

### Database migration

One Flyway migration runs automatically on startup:

- `V33` – adds a non-null `entity_name` column to `audit_log` and
  backfills historical rows with `"UNKNOWN"`.

No data loss. The migration is fast on typical CRM-sized data sets.

### Frontend / Node.js

- The frontend build now requires **Node.js 24** and **`pnpm@11.3.0`**.
  CI and the Docker build are already updated; local developer
  environments need to be bumped.
- `pnpm-workspace.yaml` is back in the repository (it was removed in
  v1.6.0) and is included in the Docker image for install consistency.

### Backend / Java

- The backend now requires **JDK 21+** at build time
  (`maven-enforcer-plugin`).
- New backend tests require a running **Docker daemon** for
  Testcontainers. Optional: enable container reuse in
  `~/.testcontainers.properties` (`testcontainers.reuse.enable=true`)
  for faster local iteration.

### Configuration changes

- Spring property prefix `openelements.search.meilisearch.*` was
  renamed to `openelements.meilisearch.*`. Operators using Spring
  property overrides must update keys; environment variables
  (`MEILI_*`) are unchanged.
- New optional env vars for the backup admin page:
  `DB_BACKUP_BASE_URL`, `DB_BACKUP_API_TOKEN`. If not set, the
  backup section displays a graceful "service not configured" state.
- New optional config block `openelements.mcp.*` for the Claude MCP
  endpoint. **Disabled by default** – do not enable in production
  without completing the GDPR / data-protection checklist documented
  in `specs/108-mcp-connector/design.md`.

### Image upload size limit

The contact photo upload limit has been raised from **2 MB to 20 MB**.
Multipart request limits in the backend have been raised accordingly.
If a reverse proxy (nginx, Traefik, …) sits in front of Open CRM,
operators should check that its own request size limit is at least
20 MB or larger.

### Authentication / SSO

If you plan to enable the Claude MCP connector, a new Authentik OAuth
client (`open-crm-mcp`, confidential, with Claude's published redirect
URI) must be created manually. Existing SSO clients are unaffected.
